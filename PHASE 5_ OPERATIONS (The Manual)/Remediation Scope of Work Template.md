## **1\. PROJECT OBJECTIVE**

## **2\. KEY DEFINITIONS (SCOPE PROTECTION)**

* **Page Template:** A unique layout or "Skeleton" code used by the CMS to render multiple URLs (e.g., the "Product Page" template may render 5,000 individual product URLs).

## **3\. TECHNICAL SCOPE OF SERVICES**

Our engineering team will manually implement the following WCAG 2.1 AA success criteria within the identified templates:

| Success Criteria | Action Item |
| :---- | :---- |
| **1.1.1 Non-text Content** | Manual hard-coding of descriptive alt attributes for all decorative and functional images. |
| **1.3.1 Info & Relationships** | Implementation of semantic HTML5 landmarks (Header, Nav, Main, Footer) and ARIA roles. |
| **2.1.1 Keyboard** | Remediation of "Keyboard Traps" to ensure full site navigation without a mouse. |
| **2.4.3 Focus Order** | Ensuring logical tabbing sequences through forms, menus, and CTAs. |
| **4.1.2 Name, Role, Value** | Correct labeling of form inputs, buttons, and dynamic interface elements. |
